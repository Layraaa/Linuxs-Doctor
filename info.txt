Información Linux's Doctor

Linux's Doctor tiene las siguientes opciones:
	- Recoger información del sistema: Recoge los datos especificados más abajo del sistema
	- Recogida de información y realizar comparaciones de evidencias: Recoge los datos especificados más abajo del sistema y los compara con un reporte anterior
	- Realizar comparaciones de evidencias: Compara dos conjuntos de evidencias
	- Información: Muestra información sobre el script
	- Salir: Sale del script

Son necesarios los siguientes archivos para que Linux's Doctor funcione. En el caso que alguno no se encuentre, se recomienda descargarlos de nuevo en el repositoria de GitHub de la herramienta inidicado al prinicipio de este documento:
	- linuxs-doctor.sh
	- analisis.sh
	- comparacion.sh
	- enviarreporte.sh
	- file.conf: Deber ser configurado si quiere guardar datos de accesso para el envio de evidencias

Linux's Doctor genera los siguientes archivos. Algunos archivos puede que no se generen/copien dependiendo del sistema operativo que estes utilizando:
	- Archivos del sistema:
		- kernelfirma.txt: Firma en sha256 del archivo del kernel del sistema
		- sudoers.txt: Archivo que incluye una lista de los usuarios que pueden usar el comando sudo para obtener privilegios de administrador
		- sudoersfirma.txt: Firma en sha256 del archivo sudoers
		- shadow.txt: Archivo que contiene una lista con nombres de usuario, contraseñas cifradas, caducidad y validez de la cuenta
		- shadowfirma.txt: Firma en sha256 del archivo shadow
		- passwd.txt:  Archivo que contiene una lista con nombres de usuario, estado de su contraseña, UID, GID, carpeta por defecto para inicio de sesión y la shell que utiliza
		- passwdfirma.txt: Firma en sha256 del archivo passwd
		- root_bash_history.txt: Archivo que contiene el historial de comandos del usuario root
		- dpkg.txt: Archivo que contiene una lista de los paquetes instalados en el sistema
		- crontab.txt: Archivo que contiene la configuración de cron
		- crontabl.txt: Archivo que contiene las tablas cron
		- grub.txt: Archivo que contiene la configuración del GRUB
		- interfaces.txt: Archivo que almacena la configuración de las interfaces de red
		- profile.txt: Archivo que contiene el primer script que se ejecuta al conectarse un usuario a una terminal de texto
		- hosts.txt: Archivo que contiene relaciones entre nombres de máquinas y direcciones IP
		- fstab.txt: Archivo que contiene la información para determinar qué opciones utilizar a la hora de montar el dispositivo especificado
		- mtab.txt: Archivo que contiene la información la lista los sistemas de ficheros ya montados y sus opciones de inicialización
		- group.txt: Archivo que contiene los nombres de los grupos y una lista de los usuarios que pertenecen a cada grupo
		- rc*.d: Contiene los scripts que se ejecutan automáticamente cuando el sistema arranca
		- init.d: Contiene todos los servicios instalados en el equipo
		- cron.d: Contiene una lista de tareas Cron para ser leídos por el servicio Cron
		- cron.hourly: Contiene los scripts que ejecuta el sistema cada hora
		- cron.dialy: Contiene los scripts que ejecuta el sistema cada día
		- cron.weekely: Contiene los scripts que ejecuta el sistema cada semana
		- cron.monthly: Contiene los scripts que ejecuta el sistema cada mes
		- cmdline.txt: Archivo que contiene los parámetros que fueron pasados al kernel en el arranque
		- blkid.txt: Archivo que muestra información sobre los dispositivos de bloque disponibles
	 	- dmesg.txt: Archivo que muestra el buffer de mensajes del núcleo
	 	- lastlog.txt: Archivo que muestra el último login realizado por los usuarios del sistema
	 	- runlevel.txt: Archivo que muestra el modo de operación del sistema operativo
		- lspci.txt: Archivo que muestra información de los buses PCI y dispositivos del sistema
		- fdisk.txt: Archivo que muestra información de las particiones de los discos duros del sistema
		- printenv.txt: Archivo que muestra las variables del sistema
		- machine_id.txt: Archivo que muestra el ID de la máquina
		- product_uuid.txt: Archivo que muestra el UUID de la máquina
		- resolv.txt: Archivo que muestra el DNS resolver
	- Servicios del sistema:
		- iptables.txt: Archivo que muestra información sobre el servicio iptables y su configuración
		- firewalld.txt: Archivo que muestra información sobre el servicio firewalld y su configuración
		- ufw.txt: Archivo que muestra información sobre el servicio ufw y su configuración
		- dhcp.txt: Archivo que muestra información sobre el servicio DHCP y su configuración
		- dns.txt: Archivo que muestra información sobre el servicio DNS y su configuración
		- vsftpd.txt: Archivo que muestra información sobre el servicio vsftpd y su configuración
		- samba.txt: Archivo que muestra información sobre el servicio samba y su configuración
		- apache.txt: Archivo que muestra información sobre el servicio apache y su configuración
		- mariadb.txt: Archivo que muestra información sobre el servicio mariadb y su configuración
		- squid.txt: Archivo que muestra información sobre el servicio squid y su configuración
		- ssh.txt: Archivo que muestra información sobre el servicio SSH y su configuración
		- php.txt: Archivo que muestra información sobre PHP y su configuración
	- Red:
		- nmcli.txt: Archivo que muestra información sobre las tarjetas de red existentes
		- netstat.txt: Archivo que muestra información sobre las conexiones del sistema
		- dig.txt: Archivo que muestra información sobre quien contesta a las peticiones DNS del sistema
		- route.txt: Archivo que muestra información sobre las tabla de enrutamiento del sistema
		- arp.txt: Archivo que muestra información sobre las tabla ARP del sistema
	- Logs del sistema:
		- auth.log*: Logs de acceso al sistema
		- dpkg.log*: Logs de estado de los paquetes del sistema
		- syslog*: Logs del sistema
		- boot.log*: Logs del inicio del sistema
		- kern.log*: Logs del kernel del sistema
		- logfile_auth_log.txt, logfile_dpkg_log.txt, logfile_syslog.txt, logfile_boot_log.txt, logfile_kern_log.txt: Todos los logs en un solo archivo
	- Logs de los servicios:
		- firewalld.txt: Logs del servicio firewalld
		- ufw*:  Logs del servicio ufw
		- vsftpd.txt: Logs del servicio vsftpd
		- log.smbd*: Logs del servicio smbd
		- log.nmbd*: Logs del servicio nmbd
		- apache2/error.log*: Logs del servicio apache2 (errores)
		- apache2/access.log*: Logs del servicio apache2 (accesos)
		- mariadb.txt: Logs del servicio mariadb
		- squid/error.log.txt: Logs del servicio squid (errores)
		- squid/access.log.txt: Logs del servicio squid (accesos)
		- logfile_ufw.txt, logfile_log.smbd.txt, logfile_log.nmbd.txt, apache2/logfile_error.log.txt, apache2/logfile_access.log.txt: Todos los logs en un solo archivo
	- Volcado de memoria la RAM
	- Analisis.txt: Contiene datos del escaneo realizado y de la maquina donde se ejecuta
	- Reporte.txt: Si se realiza una comparación de evidencias, se generará este archivo que contiene información sobre la comparación de las evidencias
